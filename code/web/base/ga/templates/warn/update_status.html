This update page might fail to load/time-out while updating => don't worry.
In that case => just refresh the page after some (~1-5) minutes.

If the update fails hard you might need to read the update logs via:
root@ga: journalctl -u ga_update.service --no-pager --full -n{LINES}

The last resort would be to manually restore the automatically created backups via:
root@ga: cd {REPO_DIR}/setup
root@ga: ansible-playbook pb_restore.yml

Default repository path: '/tmp/ga/update/{DATETIME}'
Default backup path: '/var/backups/ga/update/{DATETIME}'
